Tài liệu Kiểm Thử Người Dùng (UAT)

Tính năng: Chỉnh sửa và Tải lên Ảnh bìa (Cover Image)

1. Mục tiêu (Objective)

  • Người dùng ngoài việc cấu hình Avatar (Ảnh đại diện), giờ đây có thể tùy chỉnh hình nền Ảnh Bìa trải rộng trên đỉnh trang Hồ sơ cá nhân.
  • Tăng tính cá nhân hóa và chuyên nghiệp cho WePost.

2. Cốt truyện người dùng (User Story)

  • Là một người dùng, nhìn cái Ảnh bìa màu xám ở trang Hồ Sơ rất "tù túng và nhạt nhẽo".
  • Tôi vào mục "Chỉnh sửa hồ sơ" → Và Ồ, có một mục tải lên Ảnh Cover nằm cạnh nút tải Avatar!
  • Tôi chọn một hình phong cảnh biển lớn cực đẹp từ máy của mình, xem trước lúc nó giãn ra ngay trên web. Ấn Lưu!
  • Giao diện nẩy lại trang Hồ sơ, hình nền biển tuyệt đẹp làm tôi tự tin hẳn lên khi nói chuyện với bạn bè!

3. Phạm vi công việc (Scope)

  • Mở rộng DB: Chạy câu lệnh ALTER TABLE Users ADD CoverUrl NVARCHAR(255) DEFAULT 'default-cover.jpg'.
  • Java Backend: `User.java` thêm biến coverUrl. `UserDAO.java` đọc thêm Cột mới vào `mapResultSetToUser()`. Sửa hàm `updateProfile(..., avatarUrl, coverUrl)`.
  • Servlet `EditProfileServlet.java`: Quét qua luồng `POST` để nhặt thêm File tên coverFile (MultiPart). Sinh chuỗi thẻ Tên UUID an toàn để Save vào Server cục bộ.
  • Frontend: Biến đổi CSS .profile-cover ở `profile.jsp` từ hình chữ nhật trơn 1 màu sang thẻ HTML <img> trỏ đúng đường dẫn ảnh (giống Avatar). Sửa file JS Preview Live ở `edit-profile.jsp` để load được 2 ảnh cùng lúc.

4. Ca kiểm thử chức năng (Functional Test Cases)

  • TC01: Bấm "Chỉnh sửa hồ sơ", User upload cả 2 Ảnh cùng lúc, Form bấm Submit → Hệ thống cắt 2 tệp cho vào Thư mục và đổi mã Hash. Giao diện thay cả cặp Bìa và Avt.
  • TC02: User không Upload gì cả, ấn Lưu thôi → Servlet phát hiện dung lượng File gửi lên = 0 Bytes, nó sẽ Giữ Nguyên toàn bộ các đường dẫn ảnh hiện tại, không gán NULL vào DB.

5. Biên kiểm thử & Trường hợp Ngoại lệ (Edge/Boundary/Negative Testing)

  • Hack Quyền (Mã Độc ZIP, BASH, PDF): Tương tự thư viện Avatar, nếu File Bìa bị bắt dính ContentType ảo (Trái với image/...), Backend chặn ngang với báo lỗi, rớt request, Server an toàn.
  • Ảnh Rất Rất To: Bìa mặc định cho phép tối đa 10 MB, vượt size sẽ bị giật sập nhờ Filter Servlet. Dùng object-fit: cover trên CSS đảm bảo các bức Pixel bự không bao giờ đâm thủng màn hình điện thoại/Layout Web.

6. Tác động - Cơ sở dữ liệu và Rollback (Impact)

  • Cơ sở dữ liệu: Đây là Tính năng có CẬP NHẬT (ALTER) Bảng nên bắt buộc bạn phải dán lệnh phụ trợ vào SQL Server bằng tay.
  • Mức độ rủi ro: Trung Bình (Medium) vì CSDL có tác động Schema. Ảnh hỏng dễ gây Crash giao diện nếu quên Try-Catch DB Migration.
  • Rollback: Khóa endpoint MultiPart /edit-profile, trả lại giao diện Ảnh màu xám cố định ở thẻ class.
